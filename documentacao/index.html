Guia completo da linguagem

Programe com clareza. Em português.

PEP é uma linguagem interpretada, dinâmica e escrita em Rust. Aprenda do primeiro arquivo até templates web, WebSocket em tempo real, bancos de dados, tensores, autodiff e processamento binário.

Começar a aprenderConsultar API

01. Instalação e execução

O repositório contém o interpretador e suas dependências. Instale Rust/Cargo e compile na raiz do projeto.

cargo build --release

No Windows, o executável será target\release\pep.exe. Confira e execute:

target\release\pep.exe --versao
target\release\pep.exe exemplos\ola_mundo.pep
target\release\pep.exe

O último comando abre o REPL. Nele, use ajuda e sair. Nos próximos exemplos usamos apenas pep; coloque o executável no PATH ou use seu caminho completo.

02. Primeiro programa

Crie um arquivo ola.pep:

# Comentários começam com cerquilha
var nome = "Mundo"
imprimir("Olá, {nome}!")
pep ola.pep
# Olá, Mundo!

Uma nova linha encerra a instrução. Ponto e vírgula também é aceito, mas geralmente é desnecessário. Blocos usam chaves.

04. Controle de fluxo

Condição

se nota >= 7 {
    imprimir("Aprovado")
} senao se nota >= 5 {
    imprimir("Recuperação")
} senao {
    imprimir("Reprovado")
}

Escolha

escolher codigo {
    caso 200 { imprimir("OK") }
    caso 401, 403 { imprimir("Negado") }
    padrao { imprimir("Outro") }
}
para nome em ["Ana", "Bia"] { imprimir(nome) }

# fim inclusivo: 1, 3, 5
para i de 1 ate 5 passo 2 { imprimir(i) }

enquanto contador > 0 {
    contador -= 1
    se contador == 2 { continue }
    se contador == 0 { pare }
}
intervalo(1, 5) termina antes do 5. Já para i de 1 ate 5 inclui o 5.

05. Funções e closures

funcao saudar(nome, prefixo = "Olá") {
    retornar "{prefixo}, {nome}!"
}

funcao somar_todos(...numeros) {
    retornar reduzir(numeros, (total, n) => total + n, 0)
}

Função anônima

var dobro = funcao(n) {
    retornar n * 2
}

Função seta

var dobro = n => n * 2
var somar = (a, b) => a + b

Funções são valores e closures capturam o ambiente de definição. Parâmetros padrão são opcionais; o parâmetro ...restante deve ser o último.

06. Listas e mapas

var numeros = [1, 2, 3, 4]
numeros[0] = 10
var pares = filtrar(numeros, n => n % 2 == 0)
var quadrados = mapear(pares, n => n * n)

var pessoa = {nome: "Ana", endereco: {cidade: "Recife"}}
pessoa.idade = 28
imprimir(pessoa.nome, pessoa["idade"], pessoa.endereco.cidade)
Coleções são valores. adicionar, remover e mapa_definir retornam a coleção alterada. Atribua o resultado: lista = adicionar(lista, item).

07. Tratamento de erros

tentar {
    lancar Erro("RegraNegocio", "Operação inválida")
} capturar (erro) {
    imprimir(tipo_erro(erro), mensagem_erro(erro))
} finalmente {
    imprimir("Fim")
}

lancar aceita texto ou Erro. Consulte pilha_erro(erro). É permitido usar apenas capturar, apenas finalmente ou ambos.

08. Módulos e inclusão

Importar

importar "util.pep"
importar "math.pep" como math
imprimir(math.dobro(5))

Sem alias, símbolos públicos entram no ambiente. Com alias, ficam em um namespace.

Incluir

incluir "cabecalho.phtml"
requerer "config.pep"

incluir tolera ausência e executa cada arquivo apenas uma vez; requerer falha se ausente.

Caminhos relativos partem do arquivo atual. Pacotes ficam em pep_modules/nome/mod.pep e podem ser importados por nome. Símbolos iniciados por _ não são exportados.

09. Arquivos, JSON, CSV e texto avançado

Arquivo e JSON

var dados = {nome: "PEP", versao: 0.6}
escrever_arquivo("dados.json", json_serializar(dados))
var lido = json_deserializar(
    ler_arquivo("dados.json"))

Regex

var emails = regex_combinar_tudo(
    "[\\w.-]+@[\\w.-]+", texto)
var limpo = regex_substituir(
    "\\s+", " ", texto)

CSV

Leitura

# cada linha retorna uma lista de textos
var linhas = csv_ler("vendas.csv")
# primeira linha vira chaves do mapa
var tabela = csv_ler_mapa("pessoas.csv")
imprimir(tabela[0].nome, tabela[0].idade)

Escrita e serialização

var dados = [["nome","preco"],["Livro",39.9]]
csv_escrever("saida.csv", dados)
# parsear/serializar em memória
var parsed = csv_parsear("a,b\n1,2")
var texto  = csv_serializar(dados)

O parser CSV suporta aspas, escapes "", vírgulas dentro de campos e BOM UTF-8. Sem dependência externa.

Também há sha256, md5, hmac_sha256, Base64, datas e formatação. Não use hashes rápidos isolados para armazenar senhas.

10. Web, templates e rotas

Arquivos .phtml misturam HTML e PEP. <?pep ... ?> executa código; <?= expressão ?> escreve com escape HTML automático.

<?pep
var produtos = [{nome: "Livro", preco: 39.9}]
?>
<ul>
<?pep para produto em produtos { ?>
  <li><?= produto.nome ?> — R$ <?= produto.preco ?></li>
<?pep } ?>
</ul>
pep servir 7878 site
# http://localhost:7878

Roteamento por arquivos

A pasta informada ao comando funciona como raiz documental: index.pep atende /, produto.pep atende /produto, produto/index.pep atende /produto e produto/[id].pep atende /produto/42. Arquivos .phtml seguem as mesmas regras. No FastCGI, SCRIPT_FILENAME e DOCUMENT_ROOT do NGINX ou Apache definem a raiz.

Contexto HTTP

VariávelConteúdo
_GET / _POSTQuery string e formulário URL-encoded.
_CORPO_JSONJSON decodificado quando o Content-Type é JSON; Nulo caso contrário.
_CORPO_BRUTOCorpo original em bytes.
_ARQUIVOSUploads multipart com nome, tipo, tamanho e dados.
_CABECALHOSCabeçalhos normalizados para minúsculas.
_URL / _METODOCaminho e método.
_COOKIE / _SESSAO_IDCookies e sessão.
_PARAMSParâmetros capturados por rota.

Rotas em routes.pep

funcao detalhar() {
    json_responder({id: _PARAMS.id})
}
rota("GET", "/api/itens/:id", detalhar)

O servidor carrega routes.pep ao iniciar, e rotas têm prioridade sobre arquivos. Sessões são mantidas em memória. PEP_SESSAO_TTL define sua duração em minutos (padrão 30).

11. WebSocket

PEP implementa o protocolo WebSocket (RFC 6455) nativamente, sem dependência externa. O servidor detecta o cabeçalho Upgrade: websocket automaticamente e disponibiliza a conexão via funções ws_* na thread do handler.

rota("GET", "/ws/chat", funcao() {
    ws_aceitar()                      # handshake HTTP 101
    var id = ws_id()                  # ID único desta conexão
    ws_broadcast("#{id} entrou")
    enquanto verdadeiro {
        var msg = ws_receber()        # bloqueia até chegar frame
        se msg == nulo { pare }     # cliente desconectou
        ws_broadcast("#{id}: {msg}")
    }
    ws_broadcast("#{id} saiu")
})

Envio direto e transmissão

# enviar para uma conexão específica
ws_enviar_para(id_destino, "mensagem privada")

# listar todas as conexões ativas
var ids = ws_conexoes()
imprimir("Online: {tamanho(ids)}")

Referência das funções WebSocket

FunçãoDescrição
ws_aceitar()Realiza o handshake; obrigatório antes de qualquer outra chamada WS.
ws_receber()Bloqueia e retorna Texto, Bytes ou Nulo (desconexão).
ws_enviar(msg)Envia frame de texto para a conexão atual.
ws_enviar_bytes(bytes)Envia frame binário para a conexão atual.
ws_fechar()Envia frame Close e encerra a conexão.
ws_id()Retorna o ID único (Inteiro) da conexão atual.
ws_conexoes()Lista de IDs de todas as conexões ativas.
ws_enviar_para(id, msg)Envia mensagem para uma conexão específica por ID.
ws_broadcast(msg)Envia mensagem para todas as conexões ativas.
Cada conexão WebSocket roda em sua própria thread. ws_broadcast e ws_enviar_para permitem comunicação cruzada entre conexões sem compartilhamento de estado explícito.

12. Bancos de dados

SQLite

var db = sqlite_conectar("app.db")
sqlite_executar(db,
  "INSERT INTO pessoas(nome) VALUES (?)", ["Ana"])
var linhas = sqlite_consultar(db,
  "SELECT * FROM pessoas WHERE nome = ?", ["Ana"])
sqlite_fechar(db)

MySQL

var db = bd_conectar(
  "mysql://usuario:senha@host/banco")
var linhas = bd_consultar(db,
  "SELECT * FROM pessoas WHERE id = ?", [1])
bd_fechar(db)
Use parâmetros ?. Concatenar entrada externa em SQL permite injeção.

13. Vetores, matrizes e tensores

var a = [1, 2, 3]
imprimir(produto_interno(a, [4, 5, 6]))

var m = mat_de([[1, 2], [3, 4]])
imprimir(mat_para_lista(mat_transpor(m)))

var t = tensor_de([1, 2, 3, 4], [2, 2])
imprimir(tensor_shape(tensor_relu(t)))

Vetores usam listas. Matrizes e tensores usam representação interna eficiente com Arc<Vec<f64>> — clone de tensores é O(1). Confira as formas (tensor_shape) antes de combinar tensores.

Quantização

Reduza o uso de memória de pesos de modelos convertendo tensores para formatos compactos:

int8 assimétrico

var t = tensor_de([0.1, -0.5, 0.9], [3])
var q = tensor_quantizar_int8(t)
# q = {dados:Bytes, escala:Num, zero_ponto:Int, shape:Lista}
imprimir(q.escala, q.zero_ponto)
var orig = tensor_dequantizar_int8(q)

float16

var qf = tensor_quantizar_f16(t)
# qf = {dados:Bytes (2 bytes/elem LE), shape:Lista}
var orig = tensor_dequantizar_f16(qf)

Escala int8 assimétrica: scale=(max−min)/255. F16: 2 bytes por elemento, little-endian. Sem dependência externa.

14. Bytes, modelos e mmap

var bytes = bytes_de_texto("Olá")
imprimir(bytes_tamanho(bytes), bytes_para_hex(bytes))

var id = mmap_abrir("pesos.f32")
var pesos = mmap_tensor_f32(id, 0, 100, 64)
mmap_fechar(id)

O deslocamento de mmap é em bytes. Feche o identificador quando terminar — use finalmente para garantir. Modelos globais podem ser mantidos entre requisições com modelo_carregar e recuperados com modelo_obter.

15. IA e autodiff

PEP inclui um motor de diferenciação automática reversa (backpropagation). Crie tensores-folha com grad_tensor, execute operações rastreadas e chame grad_retropropagar na perda para acumular gradientes em todas as variáveis.

Gradiente simples

# y = x² → dy/dx = 2x; em x=3 deve dar 6
grad_ativar()
var x = grad_tensor([3.0], [1])
var y = grad_mul(x, x)
var loss = grad_soma_total(y)
grad_retropropagar(loss)
imprimir(grad_de(x))   # [6.0]
grad_limpar()

Camada linear

# y = relu(W·x) — forward + backward
grad_ativar()
var W = grad_tensor([0.5,-1.0,0.3,0.8],[2,2])
var x = grad_tensor([1.0,2.0],[2,1])
var y = grad_relu(grad_matmul(W, x))
var loss = grad_soma_total(y)
grad_retropropagar(loss)
imprimir(grad_de(W))
grad_limpar()

Referência das funções de gradiente

FunçãoDescrição
grad_ativar() / grad_desativar()Liga/desliga rastreamento de operações na fita.
grad_limpar()Descarta a fita de operações. Chame após cada iteração.
grad_zerar_todos()Zera gradientes acumulados de todos os tensores-folha.
grad_tensor(dados, shape)Cria tensor-folha rastreável (nó do grafo de computação).
grad_soma · grad_sub · grad_mul · grad_matmulOperações binárias com rastreamento na fita.
grad_relu · grad_sigmoid · grad_tanh · grad_exp · grad_log · grad_negAtivações unárias com rastreamento.
grad_soma_total(t)Redução diferenciável para escalar — ponto de partida do backward.
grad_escalar_mul(t, n)Multiplica tensor por escalar com grad rastreado.
grad_retropropagar(loss)Percorre a fita ao contrário e acumula gradientes.
grad_de(t)Retorna tensor com os gradientes acumulados de t.
grad_dados(t)Retorna os valores forward de t sem ID de gradiente.
Chame grad_limpar() após cada iteração de treinamento para liberar a fita e evitar acúmulo incorreto de gradientes entre passos.
GPU (opcional): Compile com --features gpu-cuda (NVIDIA) ou --features gpu-metal (Apple Silicon). Use tensor_para_gpu(t), tensor_para_metal(t) e tensor_para_cpu(t) para mover tensores entre dispositivos. tensor_dispositivo(t) retorna "cpu", "cuda:0" ou "metal".

18. Boas práticas

Nomes claros

Prefira total_pedido a abreviações. Divida regras em funções pequenas.

Verifique antes

Use pep --check em automações e pep formatar para consistência.

Valide entradas

Use tipo/eh_numero, escape HTML e sempre parametrize SQL com ?.

Libere recursos

Feche bancos, mapeamentos e conexões em finalmente, inclusive após falhas.

Autodiff

Chame grad_limpar() após cada passo de treinamento. Não acumule a fita entre iterações.

WebSocket

Trate o retorno nulo de ws_receber() para detectar desconexões e encerrar o loop.
